# Templates — Writing WeChat Tech Posts

## 一键总控：通用大模板 Prompt
```text
你是一个科技技术分享类微信公众号主笔与编辑总监。请为公众号读者写一篇文章。

【账号定位】{账号定位：如“绿色低碳+AI应用+工程管理”}
【目标读者】{读者画像：岗位/水平/痛点，例如“制造业节能工程师，懂基础但缺可落地方法”}
【文章主题】{主题：例如“用LLM做能耗数据异常检测的落地路线”}
【内容类型】{教程/科普/深度解读/工具评测/观点评论/案例复盘}
【目标】{让读者学会/理解/能上手/能决策/能避坑}
【长度】{1200-2000字 或 指定}
【语气】{专业但好读，略带幽默；避免说教；不油腻}
【合规要求】不编造数据与引用；不确定的内容要标注“需核实”并给出核实路径；避免夸大宣传。

输出必须包含：
1) 3-5个备选标题（含爆款风、克制风、学术风）
2) 开头：用一个“痛点场景/反常识/数据冲击”做钩子（100-150字）
3) 文章目录（3级标题，结构清晰）
4) 正文：每节包含“核心观点→解释→例子/类比→可执行步骤/清单”
5) 至少1个“踩坑清单”和1个“行动清单”
6) 配图建议：每张图画什么、放在哪一段（不需要真的画图）
7) 公众号排版建议：加粗点、金句、分隔线位置、emoji使用建议（适度）
8) 结尾：总结+互动提问+关注引导（克制、自然）
9) 30字摘要（用于转发）+ 80字摘要（用于开头导语）
```

## 钩子开头 Prompt（高阶：黄金 3 秒模板）

```text
为主题 {主题} 写 3 种高阶开头钩子：

1) 痛点场景代入式
- 核心：具体场景 + 困境回响。让读者产生“这说的就是我”的共鸣。
- 结构：[时间/场景] + [具体细节困境] + [提问/利益引导]。
- 示例：针对职场主题——“周一早上刚到公司，领导就丢来3个紧急任务，看着待办列表瞬间慌了神。其实你缺的不是努力，而是这套方法……”

2) 反差悬念吸引式
- 核心：反常识结果 + 细节落差。勾起读者“凭什么/为什么”的好奇心。
- 结构：[意外结果] + [扎心的反差细节] + [悬念揭晓引导]。
- 示例：针对理财主题——“月薪5000的同事半年存下2万块，月薪8000的你却是月光。差别就在你忽略了这3个小技巧。”

3) 真实经历分享式
- 核心：亲身经历（第一人称）+ 情绪连接。让内容具备“真实可信、值得一看”的背书感。
- 结构：[个人此前失败/普通状态] + [转变节点/现在的状态] + [价值干货预告]。
- 示例：针对写作主题——“去年写公众号阅读只有100，焦虑到失眠。直到我摸出这套‘优化法’，1个月后阅读稳在2000+。今天分享给你。”
```

## 正文段落 Prompt（观点-解释-例子-步骤）

```text
请写正文第{N}节，标题为《{小节标题}》。
输出格式固定为：
- 核心观点（1句话）
- 为什么（用通俗类比解释，<=120字）
- 例子（最好是工程/产品/业务场景）
- 怎么做（步骤清单，3-6条）
- 一句话小结（金句风但不油）
```

## 技术深度 + 公众号可读性 Prompt

```text
把{技术点：例如“RAG、蒸馏、异常检测、碳核算边界”}写成公众号可读版本：
要求：
- 先用生活类比解释（不超过80字）
- 再给严谨定义（不超过80字）
- 给一个简单示意流程（用文字画流程：A→B→C）
- 给一个常见误解与纠正
- 给一个最小可行实践（MVP）步骤
```
